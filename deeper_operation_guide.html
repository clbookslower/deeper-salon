GitHub 연동은 언제 필요한가: ① 두 대 이상의 PC에서 같은 Vault를 쓰고 싶을 때 ② "3주 전에 썼던 그 초안"을 정확히 꺼내야 할 때 ③ 노트를 팀과 공유하거나 웹에 퍼블리시할 때. 지금 단계에서는 OneDrive가 자동 백업을 해주고 있어서 불필요합니다. 3개월 뒤, 필요하다고 느끼면 그때 별도 가이드로 안내합니다.

폴더 이름, 반드시 가이드와 똑같이 해야 하나요?

기본 원칙: 자유롭게 바꿔도 됩니다. 폴더 이름은 여러분의 언어로 짓는 것이 오히려 더 잘 씁니다. "01-Grammar" 대신 "나의-재료들", "00-Daily" 대신 "매일" 이렇게 해도 전혀 문제 없습니다.

단, 딱 하나 — Templates 폴더 이름을 바꿀 경우에는 Templater 플러그인 설정도 같이 바꿔야 합니다.

경로: ⚙️ 설정 → Templater → "Template folder location"에 바꾼 폴더 이름 입력. 이것만 맞으면 나머지는 모두 자유입니다.

폴더 구조를 나중에 바꾸고 싶다면? Obsidian 안에서 폴더를 드래그해서 옮기거나 이름을 바꾸면, 그 폴더 안의 노트 링크가 자동으로 업데이트됩니다. 처음부터 완벽하게 안 잡아도 됩니다.

Obsidian에서 Claude 직접 연결 — 왜 지금은 안 되는가

Obsidian에는 Claude를 연결하는 플러그인이 존재합니다. 그런데 이 플러그인을 쓰려면 "API 키"가 필요합니다. API 키가 무엇인지부터 정확히 알아야 합니다.

claude.ai 구독 ≠ API 접근권
이 두 가지는 완전히 별개의 서비스입니다
서비스무엇인가비용
claude.ai (무료) 웹/앱에서 Claude와 대화. 지금 여러분이 쓰는 것 무료 (메시지 제한)
claude.ai Pro 더 많은 메시지, 더 긴 대화, Projects 기능 월 $20
Anthropic API 개발자용. Obsidian 플러그인·앱 등에서 Claude를 직접 호출할 때 필요한 "열쇠" 사용량만큼 별도 과금
Claude Code 터미널에서 코드·파일 작업을 Claude와 함께. API 키 사용 API 사용량 과금
핵심: claude.ai Pro를 구독해도 Obsidian 플러그인 연결에는 쓸 수 없습니다. Obsidian ↔ Claude 연결은 별도로 console.anthropic.com에서 API 키를 발급받아야 합니다.

그렇다면 지금은 어떻게 쓰나요? — 복붙이 귀찮지 않은 이유

복잡하지 않습니다. Obsidian에 쓴 것을 복사 → Claude에 붙여넣기 → Claude 답변을 복사 → Obsidian에 저장.

이 방식이 자동화보다 열등해 보일 수 있습니다. 하지만 이것은 의도적인 선택입니다. 그 이유는 다음과 같습니다.

왜 지금은 자동화보다 복붙이 맞는가
  • 내 글이 어디로 가는지 눈으로 봐야 한다. 자동화되면 "내 노트가 Claude에게 넘어갔다"는 감각이 사라집니다. 복붙하는 순간, 나는 무엇을 왜 넘기는지 의식적으로 결정합니다. 이 의식이 탐구의 질을 결정합니다.
  • 옮기는 과정이 편집 행위다. Claude 답변 전체를 복사하는 대신, "이 문장만 남기겠다"고 선택할 때 이미 여러분은 생각하고 있습니다. 자동화는 이 선택을 건너뜁니다.
  • AI가 어떻게 작동하는지 몸으로 배운다. Level 1에서 직접 손으로 맥락을 조립하는 경험이 쌓이면, Level 3-4에서 자동화 설정을 할 때 "이 플러그인이 내 대신 무엇을 하는가"가 보입니다. 도구를 이해하지 못한 채 자동화부터 시작하면, 오류가 생겼을 때 왜 그런지 알 수 없습니다.

확장 경로 — 레벨을 하나씩 올려야 하는 이유

AI 도구를 처음부터 가장 강력한 버전으로 쓰는 것이 왜 오히려 손해인지 논리적으로 설명합니다.

핵심 원칙: 도구는 내가 무엇을 원하는지 알 때 비로소 유용합니다. 내가 무엇을 원하는지 모르는 상태에서 강력한 도구를 쥐면, 도구가 방향을 결정합니다. 그것은 자동화가 아니라 위임입니다.
LEVEL 1 — 지금 (무료 Claude)

claude.ai 웹/앱 ↔ Obsidian 수동 복붙. 설정 없이 바로 시작.

이 단계에서 배우는 것: Claude에게 어떤 맥락을 줄 때 좋은 답이 오는지, 어떤 질문 방식이 나에게 맞는지. 이것을 체득하지 않으면 이후 자동화가 아무 의미가 없습니다.

LEVEL 2 — Claude Pro (월 $20)

메시지 제한 해제, Projects 기능으로 대화 맥락이 대화 간 유지됩니다. "나 소개 카드"를 매번 붙여넣지 않아도 됩니다.

언제 올라가면 좋은가: 무료 메시지 한도에 자주 걸려 탐구가 끊기기 시작할 때. 보통 3-4주 차에 그 시점이 옵니다.

👉 claude.ai → 왼쪽 사이드바 → Upgrade to Pro

LEVEL 3 — Anthropic API + Obsidian 플러그인

Obsidian 안에서 바로 Claude를 호출합니다. 노트를 열어두고 "이 노트 분석해줘"가 가능해집니다.

언제 올라가면 좋은가: Level 1-2에서 "내가 Claude에게 자주 하는 요청의 패턴"을 파악했을 때. 무엇을 자동화할지 알아야 자동화가 의미 있습니다. 6월쯤 시도하기에 적당합니다.

  • console.anthropic.com → API Keys → Create key
  • Obsidian 플러그인 "Smart Connections" 또는 "Copilot" 설치
  • 현재 Vault 구조 그대로 사용 가능
LEVEL 4 — Claude Code

터미널에서 "내 Vault 전체를 분석해서 원천 텍스트 초안 써줘"가 가능해집니다. Obsidian 파일을 직접 읽고 편집합니다.

언제 올라가면 좋은가: 7월 이후, 콘텐츠를 반복 생산하는 루틴이 잡혔을 때. 루틴 없이 Claude Code부터 쓰면 강력한 도구가 방향 없이 작동합니다. API 키 필요, 터미널 기초 필요. 현재 OneDrive + 폴더 구조와 완전 호환.

결론: 레벨을 건너뛰는 것은 문법도 모르는 채 소설을 쓰는 것과 같습니다. 각 레벨에서 배우는 것이 다음 레벨의 기초가 됩니다. 지금 설정한 Vault 구조는 어떤 방향으로 확장해도 그대로 씁니다.

무료 Claude, 더 잘 쓰는 법

메시지 제한 팁: 짧은 확인 질문 여러 번보다 "질문 + 내 생각 + 원하는 것"을 한 번에 붙여넣어 한 대화에서 깊게 파는 것이 훨씬 효율적입니다. 하루 Claude 탐구는 3-5회면 충분합니다.
대화 기록 문제: 무료 Claude는 새 대화창을 열면 이전 대화를 기억하지 못합니다. 아래 "나 소개 카드"를 매번 첫 메시지에 붙여넣으면 Claude가 즉시 맥락을 파악합니다.

나 소개 카드 — Obsidian에 저장해두고 매번 붙여넣기

나는 [직업/역할]이고, [주요 경험 1-2줄]을 해온 사람이야. 지금 '디퍼 언브랜딩살롱'이라는 3개월 프로그램에 참여 중이야. 목표는 나만의 원천 텍스트(나만이 말할 수 있는 이야기)를 발굴하고, 그것을 지속 가능한 콘텐츠 비즈니스로 구조화하는 거야. 오늘 탐구하고 싶은 질문은: [질문] 내가 지금 생각하는 것은: [내 생각 2-3문장] 이 탐구를 도와줘. 정답보다는 더 깊은 질문으로 이끌어줘.

이 카드를 Templates 폴더에 Claude-나소개카드.md로 저장해두세요. 새 대화마다 열어서 복붙합니다.

Day 1 — 오늘

오늘 할 일

GitHub · OneDrive · Obsidian 계정 가입 완료. 이제 PC에서 Obsidian을 설치하고, OneDrive와 연결하고, 오늘 밤 첫 글을 씁니다.

⚠️ 반드시 PC(Windows 또는 Mac)에서 진행하세요.
아이패드·아이폰에서 Obsidian을 열면 Vault 위치를 직접 지정하는 "Browse" 버튼이 없고, iCloud 저장만 선택됩니다. OneDrive와 연결하려면 데스크탑 앱(PC/Mac)이 필요합니다.

아이패드는 Vault 설정이 완료된 이후에 열람·가벼운 메모용으로 쓸 수 있습니다. 세팅은 PC에서 먼저.
오늘의 목표: PC에 Obsidian을 설치하고, OneDrive 안에 Vault를 만들고, 폴더 구조와 Daily Note 템플릿을 설정합니다. 오늘 밤 첫 질문을 씁니다. GitHub 연동은 하지 않아도 됩니다 — OneDrive가 자동으로 모든 파일을 클라우드에 백업합니다.

STEP 0 — PC에 Obsidian 설치하기

Obsidian은 무료 앱입니다. 공식 사이트에서 직접 다운로드합니다.

  • 브라우저에서 obsidian.md 에 접속합니다

    가운데 보라색 "Get Obsidian for free" 또는 "Download" 버튼을 클릭합니다.

    💡 앱스토어가 아닌 공식 사이트에서 받는 이유: 앱스토어 버전은 기능이 일부 제한될 수 있습니다. 공식 사이트 버전이 플러그인 등 모든 기능을 지원합니다.
  • 운영체제에 맞는 설치 파일을 선택합니다

    Windows: "Download for Windows (64-bit installer)" 클릭 → 다운로드된 Obsidian-설치파일.exe 더블클릭 → "다음" 계속 클릭 → 설치 완료

    Mac: "Download for macOS" 클릭 → 다운로드된 .dmg 파일 더블클릭 → Obsidian 아이콘을 Applications 폴더로 드래그

    ✅ 설치 후 Obsidian을 열면 "Open vault" 화면이 나타납니다. 이 화면이 보이면 성공입니다.

STEP 1 — PC에 OneDrive 데스크탑 앱 확인하기

OneDrive 계정은 이미 만들었습니다. 이제 PC에서 OneDrive 폴더를 로컬에서 사용할 수 있도록 데스크탑 앱이 실행 중인지 확인합니다.

  • OneDrive 데스크탑 앱이 실행 중인지 확인합니다

    Windows: 화면 오른쪽 아래 작업표시줄 트레이 영역(시계 근처)에 흰색 또는 파란색 구름 ☁️ 아이콘이 보이면 이미 실행 중입니다.

    Mac: 화면 위 메뉴바 오른쪽에 구름 ☁️ 아이콘이 보이면 실행 중입니다.

    💡 아이콘이 없다면: OneDrive 다운로드 페이지에서 데스크탑 앱을 설치하고, 계정으로 로그인합니다.
  • PC에서 OneDrive 폴더가 보이는지 확인합니다

    Windows: 탐색기(폴더 열기)를 열면 왼쪽 목록에 "OneDrive" 항목이 있습니다. 클릭하면 내 OneDrive 파일들이 보입니다.

    Mac: Finder를 열면 왼쪽 사이드바에 "OneDrive"가 있습니다.

    ✅ OneDrive 폴더가 보이면 다음 단계로 이동합니다. 이 폴더가 PC와 클라우드를 자동으로 연결해줍니다.
  • OneDrive 폴더 안에 새 폴더를 만듭니다. 이름: Deeper Salon

    Windows: 탐색기에서 OneDrive 폴더 안 빈 곳을 우클릭 → "새로 만들기" → "폴더" → 이름 입력
    Mac: Finder에서 OneDrive 폴더 안 빈 곳을 우클릭 → "새로운 폴더" → 이름 입력

STEP 2 — Obsidian Vault를 OneDrive 안에 생성하기

"Vault"는 Obsidian이 노트를 저장하는 폴더입니다. 쉽게 말해 Obsidian의 집입니다. 이 집을 방금 만든 OneDrive/Deeper Salon 폴더 안에 만들면, 모든 노트가 자동으로 클라우드에 백업됩니다.

  • Obsidian을 열면 "Open vault" 화면이 나타납니다

    이미 다른 Vault가 있다면, 왼쪽 아래 금고(🏛️) 아이콘을 클릭하면 Vault 관리 화면으로 돌아갈 수 있습니다.

  • "Create new vault"를 클릭합니다

    새 Vault를 만드는 옵션입니다.

    • Vault name: deeper-salon 이라고 입력합니다 (한글도 됩니다: "디퍼살롱")
    • Location: "Browse" 버튼을 클릭합니다 → 탐색기(또는 Finder)가 열리면, OneDrive → Deeper Salon 폴더를 찾아 선택합니다
    • 중요: iCloud나 Documents 등 다른 폴더가 아닌 OneDrive/Deeper Salon이 선택됐는지 확인 후 "Create" 클릭
  • Obsidian이 열리면 왼쪽에 파일 탐색기가 보입니다

    여기가 여러분의 첫 Vault입니다. 지금은 비어있지만, 3개월이 지나면 여러분의 가장 귀한 저장소가 됩니다.

    ✅ 확인: OneDrive 폴더를 열어보세요. Deeper Salon 폴더 안에 deeper-salon 폴더가 생겼다면 성공입니다.

STEP 3 — 폴더 구조 만들기

Obsidian 왼쪽 파일 탐색기에서 아래 폴더들을 만듭니다. 빈 곳에서 우클릭 → "New folder"를 선택하면 됩니다.

폴더가 왜 이렇게 나뉘어 있나요?
5월에 쌓은 "01-Grammar" 데이터가 6월의 "02-Logic" 구조 설계의 재료가 됩니다. 7월의 "03-Rhetoric" 발행물은 이 두 달의 결과물입니다. 이 구조 자체가 3개월 여정의 흐름을 반영합니다.

STEP 4 — 플러그인 3개 설치하기

⚠️ 플러그인 설치는 PC(데스크탑 앱)에서만 가능합니다. 아이패드·아이폰용 Obsidian 앱은 커뮤니티 플러그인을 지원하지 않습니다. 반드시 PC에서 진행하세요.

플러그인은 Obsidian에 새 기능을 추가하는 확장 프로그램입니다.

왜 플러그인이 필요한가?
기본 Obsidian은 텍스트 파일을 저장하는 도구입니다. 하지만 우리에게는 두 가지가 더 필요합니다.

① 날짜 자동 입력 — 매일 노트를 열 때 오늘 날짜, 요일이 자동으로 채워지게 하려면 Templater가 필요합니다. 직접 타이핑하면 실수가 생기고, 날짜 포맷이 달라져 나중에 기록을 검색하기 어렵습니다.

② 하루 한 번 클릭으로 새 노트 생성 — Daily Notes 기능이 없으면 매번 폴더를 찾아가 노트를 직접 만들어야 합니다. 이 마찰이 매일 쓰는 습관을 방해합니다.

③ 나중에 패턴 조회 — Dataview가 있으면 "5월에 내가 쓴 질문 전체 모아보기"가 한 줄의 쿼리로 됩니다. 지금 당장 쓰지 않아도 데이터가 쌓인 뒤에 유용합니다.
플러그인 설치 경로 (영문 / 한글 UI 동일):
왼쪽 하단 ⚙️ 아이콘 클릭 → 왼쪽 메뉴에서 "Community plugins" (한글: "커뮤니티 플러그인") 클릭 → "Browse" (한글: "검색") 버튼 클릭 → 검색창에 플러그인 이름 입력 → "Install" 클릭 → "Enable" (한글: "활성화") 클릭

처음 Community plugins에 들어가면 "Safe mode" 경고가 뜰 수 있습니다. "Turn off Safe mode" (한글: "안전 모드 비활성화") 버튼을 클릭해야 플러그인을 설치할 수 있습니다.
① Templater — 스마트 템플릿
날짜 자동 입력, 폴더 지정 등 노트 작성을 도와주는 플러그인

설치 후 추가 설정이 필요합니다:

  • ⚙️ 설정 → Templater 항목 찾기 → "Template folder location"에 Templates 입력 Templates 폴더를 플러그인이 인식하도록 알려주는 설정입니다
  • "Trigger Templater on new file creation" 토글 켜기 (파란색이 되면 활성화) 새 노트를 만들 때 자동으로 템플릿이 적용되게 합니다
② Daily Notes — 일간 노트 자동 생성 (기본 내장)
매일 오늘 날짜로 새 노트를 자동 생성해주는 기능. 설치 없이 활성화만 하면 됩니다.
  • ⚙️ 설정 → "Core plugins" → "Daily notes" 찾아서 토글 켜기
  • Daily notes 설정으로 들어가서: 설정 항목이 왼쪽 메뉴에 나타납니다
  • "New file location": 00-Daily 입력 새 일간 노트가 저장될 폴더를 지정합니다
  • "Date format": YYYY-MM-DD 입력 파일 이름이 "2026-05-07" 형식이 됩니다
  • "Template file location": 다음 단계에서 템플릿 만든 후 여기 경로를 입력합니다 지금은 비워두세요. 5단계 완료 후 다시 돌아옵니다.
③ Dataview — 노트 데이터 조회
나중에 "내가 쓴 모든 질문 모아보기", "이번 달 완료한 것 목록" 같은 기능을 씁니다. 지금은 설치만.
  • 검색해서 설치하고 "Enable"만 클릭. 지금 당장 별도 설정 없음

STEP 5 — Daily Note 템플릿 만들기

왜 템플릿이 필요한가?
매일 노트를 열 때마다 빈 화면을 마주하면 "무엇부터 써야 하지"라는 마찰이 생깁니다. 이 작은 마찰이 쓰는 습관을 방해합니다. 템플릿은 이 마찰을 없앱니다 — 노트를 열면 오늘 날짜와 섹션들이 이미 준비돼 있고, 그냥 쓰기 시작하면 됩니다.

또한 3개월 뒤 "내가 5월에 매일 무슨 질문을 썼는지" 한번에 보려면 노트들이 같은 구조여야 합니다. 템플릿이 그 구조를 보장합니다.
아래 내용을 복사해서 붙여넣는 것의 의미:
<% tp.date.now("YYYY-MM-DD") %> 이 코드가 있으면, 노트를 열 때 Templater 플러그인이 이것을 오늘 날짜(예: 2026-05-04)로 자동 교체합니다. 직접 타이핑하는 게 아니라 "이 자리에 오늘 날짜를 넣어줘"라는 지시를 코드로 써두는 것입니다.

---로 감싼 상단 구역은 날짜·유형 정보를 Obsidian이 데이터로 인식하는 구역(frontmatter)입니다. 글쓰기를 방해하지 않으며, 나중에 "type: daily인 노트 전체 조회" 같은 검색을 할 때 씁니다.

한 번만 만들어두면 매일 씁니다.

  • Obsidian 왼쪽 파일 탐색기에서 "Templates" 폴더를 찾습니다

    앞에서 만든 폴더입니다. "Templates" 폴더 위에 마우스를 올리면 오른쪽에 아이콘들이 나타납니다. 그 중 종이 + 펜 모양의 아이콘(New note)을 클릭합니다.
    또는: "Templates" 폴더를 우클릭 → "New note" 클릭

  • 새로 생긴 파일 이름을 Daily-Template으로 입력합니다

    파일이 생성되면 이름 입력 상태로 시작합니다. 바로 "Daily-Template"을 타이핑하고 Enter를 누릅니다. 이미 이름이 지정된 상태라면, 파일명을 클릭하거나 F2를 눌러 수정합니다.

  • 오른쪽 편집 화면에 아래 내용을 그대로 복사해서 붙여넣습니다

    붙여넣기 단축키: Windows = Ctrl+V, Mac = Cmd+V

    이 템플릿에서 <% ... %>로 감싼 부분은 Templater 플러그인이 처리하는 코드입니다.
    예: <% tp.date.now("YYYY-MM-DD") %>는 노트를 열 때 자동으로 오늘 날짜(예: 2026-05-07)로 바뀝니다.
    이 부분은 건드리지 않아도 됩니다. ---로 감싼 상단 영역(frontmatter)은 Obsidian이 내부적으로 날짜·종류 등을 인식하는 데이터 구역입니다. 내용이 보이기는 하지만 글쓰기에 방해되지 않습니다.
    --- date: <% tp.date.now("YYYY-MM-DD") %> week: <% tp.date.now("W") %> type: daily --- # <% tp.date.now("MM월 DD일 (ddd)") %> ## 오늘의 질문 > ## 모닝페이지 메모 (오늘 쓴 것 중 한 줄만 남긴다) ## 오늘의 키씬 — 가장 중요한 한 가지 - [ ] ## Claude와 탐구한 것 (질문을 가져가서 나눈 것 중 남길 문장) ## 오늘 발견한 것 ## 감사 + 비전 - 감사: - 비전:

    ⚠️ 붙여넣기 후 내용이 이상하게 보일 수 있습니다. 이것은 정상입니다. <% %> 부분이 아직 처리되지 않은 상태입니다. 실제 Daily Note를 열 때 자동으로 날짜로 바뀝니다.

  • Daily Notes 설정에 이 템플릿을 연결합니다

    ⚙️(설정) → 왼쪽 메뉴에서 "Daily notes" 클릭 → "Template file location" 항목에 아래를 입력합니다:

    Templates/Daily-Template

    앞에서 "지금은 비워두세요"라고 안내했던 그 항목입니다. 이제 채울 수 있습니다.

    ✅ 확인: 입력 후 다른 곳을 클릭하면 자동 저장됩니다. 별도 저장 버튼이 없습니다.
  • Templater 설정도 이 폴더와 연결합니다

    ⚙️(설정) → 왼쪽 메뉴에서 "Templater" 클릭 → "Template folder location"에 아래를 입력합니다:

    Templates

    Daily Notes와 Templater, 두 곳 모두에 연결해야 날짜 자동 입력이 작동합니다.

템플릿이 적용이 안 되거나 날짜가 안 바뀐다면?
두 가지를 확인하세요.
① ⚙️ 설정 → Community plugins → Templater가 활성화(Enable) 상태인지 확인
② 수동으로 적용: 노트를 열고 Ctrl+P (Mac: Cmd+P) → "Templater: Open Insert Template modal" 입력 → "Daily-Template" 선택

STEP 6 — 첫 Daily Note 열기 및 확인

  • Obsidian 왼쪽 사이드바에서 📅 달력 아이콘을 클릭합니다

    오늘 날짜 이름(예: 2026-05-07.md)의 새 노트가 열립니다. 상단에 오늘 날짜가 자동으로 채워져 있고, 아래 섹션들이 보이면 성공입니다.

    달력 아이콘이 보이지 않는다면: Ctrl+P (Mac: Cmd+P) → "Open today's daily note" 입력 후 Enter

    ✅ "## 오늘의 질문" 섹션이 보인다면 모든 설정이 완료된 것입니다. 이제 오늘의 질문을 적을 준비가 됐습니다.

STEP 7 — 오늘 밤: 첫 질문 쓰기

오늘의 진짜 미션은 여기입니다. 설치는 도구입니다. 글이 본체입니다.

Daily Note의 "오늘의 질문" 섹션에 아래 질문을 붙여넣고, 먼저 혼자 2-3분 생각해본 뒤 Claude에게 가져가세요.

오늘의 질문 — 5/4

"내가 가장 가슴 뛰었던 일은 무엇이고, 그 이유는?"

Claude에서 이렇게 탐구하세요

claude.ai 를 열고, 위에서 만든 "나 소개 카드"를 붙여넣은 뒤 아래처럼 대화를 시작하세요.

복사해서 쓰는 첫 탐구 프롬프트
나는 [직업 또는 지금 하는 일/역할]이야. [한 줄 소개]. 지금 '디퍼 언브랜딩살롱'이라는 프로그램에서 나만의 원천 텍스트를 발굴하는 중이야. 오늘 탐구할 질문: "내가 가장 가슴 뛰었던 일은 무엇이고, 그 이유는?" 내가 지금 떠오르는 것은: [내 생각 2-3문장 자유롭게] 이 질문을 더 깊이 파고들 수 있도록 도와줘. 정답을 주는 게 아니라, 내가 더 깊이 생각할 수 있는 질문 3개를 던져줘.
탐구 결과를 Obsidian에 저장하는 법: Claude의 답변 중 "이 질문이다" 싶은 것, 처음 보는 각도, 내 안에서 울린 문장을 Daily Note의 "Claude와 탐구한 것" 섹션에 붙여넣고 한 줄 메모를 덧붙이세요. 이 반복이 쌓이면 5월 말에 원천 텍스트가 됩니다.

PC 셋업 완료 후 — 아이패드·아이폰에서 열어보기

PC에서 Vault 셋업이 완료되면 아이패드나 아이폰에서도 노트를 열람하고 간단히 메모할 수 있습니다. 단, Vault를 새로 만들거나 플러그인을 설치하는 작업은 항상 PC에서 합니다.
  • 아이패드에 Obsidian 앱 설치

    App Store에서 "Obsidian" 검색 후 설치합니다. 무료입니다.

  • 아이패드에 OneDrive 앱 설치 및 로그인

    App Store에서 "Microsoft OneDrive" 설치 후 PC와 같은 계정으로 로그인합니다.

  • 아이패드 Obsidian은 iCloud 기반으로만 Vault를 엽니다

    아이패드용 Obsidian은 OneDrive 경로를 직접 지정할 수 없습니다. 가장 간단한 해결법은 이동 중 메모는 아이패드의 메모 앱이나 클로드 앱에 임시로 적은 뒤 → 집에 와서 PC Obsidian에 옮기는 방식입니다. 별도 설정 없이 바로 쓸 수 있어서 실용적입니다.

    📱 실용적인 흐름: 세션 중·이동 중 → 아이패드 메모 앱에 빠르게 기록 → 저녁에 PC Obsidian Daily Note로 옮기며 정리.

GitHub 연동 — 선택 사항 (지금 안 해도 됩니다)

왜 지금 하지 않아도 되는가: OneDrive가 이미 자동으로 모든 파일을 클라우드에 백업합니다. 개발자가 아니라면 지금 단계에서 GitHub 연동보다 첫 글 쓰는 것이 백 배 중요합니다.

GitHub 연동이 필요하다고 느껴질 때 그때 별도 가이드로 안내드립니다.

매일의 흐름 — 3단계

  • Obsidian에서 오늘 질문 적기

    Daily Note를 열고 "오늘의 질문" 섹션에 질문을 씁니다. 제공된 질문을 쓰거나, 스스로 떠오른 것을 씁니다. 먼저 2-3분 혼자 생각합니다.

  • Claude에서 탐구

    나 소개 카드 + 질문 + 내 생각을 함께 붙여넣어 대화를 시작합니다. 이 순서가 중요합니다. 질문만 던지면 일반적인 답이 오지만, 내 맥락과 내 생각을 먼저 주면 Claude가 나에게 맞춘 질문으로 답합니다.

  • 인사이트를 Obsidian에 저장

    Claude 대화에서 "이거다" 싶은 문장, 새로운 각도, 이번에 처음 알게 된 내 패턴을 Daily Note의 "Claude와 탐구한 것" 섹션에 붙여넣습니다. Claude 말을 그대로 복붙하지 말고, 내 말로 한 문장씩 정리하는 것이 핵심입니다.

이렇게 쓰면 안 됩니다

❌ 나쁜 예: "나의 독점적 관점을 찾아줘"
→ Claude는 나를 모릅니다. 재료 없이 결과를 달라는 요청은 일반적이고 표면적인 답만 줍니다.
✓ 좋은 예: "나는 10년 동안 [분야]를 해왔고, 최근에 [경험]을 했어. 이 경험에서 내가 계속 마음에 걸리는 것은 [내용]이야. 이 불편함이 나만의 관점으로 발전할 수 있는지 탐구하고 싶어. 더 깊은 질문 3개 던져줘."
→ 재료를 주면 Claude가 그것을 재료로 탐구를 도와줍니다.

복붙 가능한 탐구 프롬프트 모음

상황에 맞게 골라 쓰세요. [ ] 안은 직접 채워넣으세요.

글 패턴 분석용 (목소리 지문 찾기)

아래는 내가 가장 자연스럽게 쓴 글들이야: [글 1 붙여넣기] --- [글 2 붙여넣기] 이 글들에서 나만의 언어 패턴을 찾아줘. 자주 쓰는 단어, 문장 구조, 독특한 표현 방식, 감정이 올라오는 주제 등을 분석해줘. "이 사람은 이런 사람이다"가 드러나는 부분을 짚어줘.

독점적 관점 탐구용

내 경험 조합이야: - [경험 A: 예) 10년간 브랜드 디자인] - [경험 B: 예) 요가 수련 5년] - [경험 C: 예) 싱글맘으로 창업] 내 분야에서 다들 당연히 여기지만 나만 의문을 품는 것: [내용] 이 조합에서 아무도 나 대신 말할 수 없는 관점의 가능성을 5가지 방향으로 탐구해줘. 그 중 가장 희소성 있어 보이는 것에 별표(*) 표시해줘.

월간 패턴 리뷰용 (매달 마지막 주)

지난 한 달 동안 내가 매일 탐구한 질문들이야: [질문 목록 붙여넣기 — 각 날짜의 "오늘의 질문" 섹션을 모아서] 그리고 내가 메모한 인사이트들: [주요 발견 붙여넣기] 이 한 달의 탐구에서: 1. 반복적으로 나타나는 주제나 단어가 있다면? 2. 아직 탐구되지 않은 공백이 보인다면? 3. 이것이 나만의 관점으로 발전한다면 어떤 방향이 가장 유망한가?

5월 매일 질문

주말을 제외한 평일마다 하나씩 배정했습니다. 그날의 질문을 Daily Note에 옮겨 적고 Claude와 탐구합니다. 각 질문은 5월 안에 원천 텍스트를, 6월의 구조 설계를, 7월의 발화를 위한 밑그림을 동시에 그립니다.

질문을 대하는 방법: 좋은 답을 찾으려 하지 마세요. 질문이 불편하거나 막막하다면 그것이 신호입니다. 그 불편함에서 쓰세요. 3줄이면 충분합니다.

Session Guide

세션을 최대로 활용하는 법

줌 세션은 매주 목요일 밤 10시입니다. 세션 안에는 각자 이번 주 발견한 것을 나누는 인사이트 공유회가 포함됩니다. 준비한 사람과 그렇지 않은 사람의 차이는 공유회에서 곧 드러납니다.

용어 정리: 세션 = 매주 목요일 밤 10시 줌 모임 전체. 인사이트 공유회 = 세션 안에서 각자 이번 주 탐구에서 발견한 것을 짧게 나누는 시간. 발표라기보다 "이번 주 나에게 걸린 것"을 꺼내놓는 자리입니다.
세션 전 (30분 — 수요일 저녁 권장)
  • 이번 주 Daily Note 5개를 훑으며 반복 등장한 키워드 찾기
  • 인사이트 공유회 준비: 이번 주 탐구에서 가장 기억에 남는 발견 한 줄 정리 — "오늘 말하지 않으면 아쉬울 것"
  • 아직 말하지 못한 것, 막히는 부분 메모
세션 중
  • Obsidian 새 노트를 열어두고 실시간 기록 — 완성된 문장 아니어도 됩니다
  • 마음에 오래 남거나 자꾸 눈이 가는 단어·문장 하나 메모 — 이것이 이번 주 탐구 소재가 됩니다
  • 다른 멤버의 인사이트 공유에서 내 안에 울린 것도 기록합니다
세션 후 24시간 안에
  • 세션 노트 정리 → 해당 주차 폴더에 저장
  • 마음에 남은 그 단어·문장을 Claude에게 가져가 탐구 — "이것이 왜 자꾸 눈에 밟히는가?"
  • 다음 주 인사이트 공유회 준비 — 다음 주 탐구 방향 한 줄 잡기
  • 디퍼 트래커 앱에서 세션 참여 XP 체크
인사이트 공유회의 본질: 세션에서 들은 것보다 내가 말한 것이 더 오래 남습니다. 완벽한 인사이트를 준비할 필요 없습니다. "이번 주 이 질문에서 이런 게 걸렸다"는 한 문장이면 충분합니다.
Milestones

3개월의 이정표

마일스톤은 성과 지점이 아니라 점검 지점입니다. 여기서 멈추고 돌아보는 것이 다음 단계를 단단하게 만듭니다.

4월 23일 (목) — 완료
ORIENTATION — 출발
나는 어디서 길을 잃었는가. 말하는 것만으로 이미 무언가 움직이기 시작했습니다.
오늘 — 지금
시스템 셋업 + 첫 질문
OneDrive·Obsidian 연결 완료. 첫 Daily Note에 첫 질문을 씁니다. 이것이 진짜 시작입니다.
5월 21일 (수)
독점적 관점 첫 명명
어색해도 됩니다. "나만이 말할 수 있는 것"을 처음으로 문장으로 씁니다.
5월 28일 (수) ★
GRAMMAR 완료 — 원천 텍스트 v1
4주간 발굴한 것들을 한 장으로 모읍니다. 이후 모든 것의 기반. 서로의 초안을 소리 내어 읽어주는 것으로 마무리합니다.
6월 11일 (수)
IP 확장 지도 초안
나의 원천 텍스트가 글→책→강의→커뮤니티로 어떻게 확장될 수 있는지 지도 초안.
6월 25일 (수) ★
LOGIC 완료 — 나만의 지도 v1
원천 텍스트 + IP 구조 + 창작 루틴을 통합한 나만의 지도 1차 버전. 완성이 아닌 시작입니다.
7월 2일 (수)
첫 콘텐츠 발행
어떤 채널이든, 어떤 형식이든 — 세상에 처음으로 내놓습니다.
7월 16일 (수)
RHETORIC 완료 + 다음 분기 계획
3개월 총체적 점검. 다음 분기 실행 계획 1페이지 완성.
7월 25일 (토) 오후 3시
오프라인 모임 — 3개월 마무리
해체하고, 다시 읽고, 설계하고, 실험한 3개월. 각자가 얼마나 멀리 왔는지를 서로가 가장 잘 알고 있습니다.

7월 25일 전에 준비하면 좋은 것

  • 원천 텍스트 최종 버전 (A4 1장)
  • 3개월 Obsidian 노트 중 가장 중요한 발견 3가지 정리
  • 실제 발행한 콘텐츠 중 하나 (어떤 형식이든)
  • 다음 3개월 실행 계획 한 페이지
  • 나에게 쓰는 편지 — "3개월 전의 나에게"
