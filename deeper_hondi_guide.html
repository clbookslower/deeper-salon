Setup

지금 당장 셋업

멤버 가이드의 기본 셋업 위에 혼디 레이어를 추가합니다. 플러그인·템플릿은 동일하고, 폴더 구조와 Claude Projects가 다릅니다.

먼저 확인: 멤버 가이드의 STEP 0~6(Obsidian 설치, OneDrive 연결, 플러그인 설치, 템플릿 만들기)을 완료했다면 이 가이드의 STEP 1(폴더 추가)부터 시작합니다. 아직 안 했다면 멤버 가이드 STEP 0부터 먼저 진행합니다.

STEP 1 — Vault에 혼디 폴더 추가

기존 deeper-salon Vault 안에 HONDI폴더를 추가합니다. Obsidian 왼쪽 파일 탐색기에서 빈 곳 우클릭 → New folder.

deeper-salon/ ├── 00-Daily/ ← 나의 일간 노트 (변경 없음) ├── 01-Grammar/ ← 나의 5월 탐구 (변경 없음) ├── 02-Logic/ ← 나의 6월 구조 (변경 없음) ├── 03-Rhetoric/ ← 나의 7월 발화 (변경 없음) ├── Templates/ ← 템플릿 (변경 없음) │ └── HONDI/ ← 지금 새로 만드는 폴더 ├── 01-원천텍스트/ ← 혼디의 철학·언어·핵심 주제 ├── 02-집필/ ← 책 목차, 챕터 초안 ├── 03-강의/ ← 강의 커리큘럼, 슬라이드 메모 ├── 04-기획제안/ ← 제안서 초안, 피치 덱 메모 ├── 05-자료/ ← 리서치, 참고 문헌, 데이터 ├── 06-업무/ ← 회의록, 할일, 진행 현황 └── 협업-이경선/ ← 이경선과 공유할 내용 스테이징

STEP 3 — 이경선과 공유 폴더 만들기

OneDrive에 공유 폴더를 만들어 이경선 씨와 연결합니다.

  • OneDrive 웹(onedrive.live.com)에 접속합니다

    루트 폴더에 혼디 × 디퍼 협업 폴더를 새로 만듭니다.

  • 폴더 공유 설정

    폴더 우클릭 → "공유" → "링크 설정" → "특정 사용자"로 설정 → 이경선 씨 이메일 입력 → 편집 권한 부여 → 공유 링크 전송

    ✅ 이 폴더가 두 사람의 협업 허브가 됩니다. 두 사람 모두 PC에서 이 폴더가 OneDrive 로컬 폴더로 동기화됩니다.
  • 공유 폴더 안 구조 잡기

프로젝트 ① — 나의 탐구 디퍼 멤버
일간 질문 탐구, 원천 텍스트 발굴 전용
나는 야간비행이야. 제주에서 '혼디' 프로젝트를 운영하는 대표이사이고, 동시에 '디퍼 언브랜딩살롱 1기' 멤버로 나만의 원천 텍스트를 발굴하는 중이야. 지금은 5월 — Grammar 단계. 매일 질문 하나를 깊이 탐구하는 중이야. 내 탐구에서 나오는 것들이 혼디 프로젝트의 언어와 철학의 기반이 될 거야. 도움을 요청할 때: - 정답보다 더 깊은 질문으로 이끌어줘 - 내 언어 패턴이나 반복 주제가 보이면 짚어줘 - 혼디와 연결되는 지점이 보이면 자연스럽게 언급해줘
프로젝트 ② — 혼디 프로젝트 혼디 CEO
집필, 강의, 기획 제안, 업무 전반
나는 제주 '혼디' 프로젝트 대표이사 야간비행이야. '혼디'는 제주어로 "함께"라는 뜻이야. 현재 진행 중인 것들: - 트리비움(문법→논리→수사) 기반 혼디 집필 - 강의 커리큘럼 설계 - 기획 제안서 작성 - 프로젝트 운영 업무 이 프로젝트에서 도움을 요청할 때: - 트리비움 프레임워크를 항상 기억해줘: 문법(해체·수집) → 논리(구조·설계) → 수사(발화·설득) - 제주라는 로컬리티와 공동체적 가치가 혼디의 핵심이야. 이것이 흐려지지 않도록 잡아줘 - 기획 제안서를 쓸 때는 설득력과 나다움이 동시에 살아있도록 도와줘
프로젝트 ③ — 혼디 × 이경선 협업 협업
이경선과 함께 진행하는 작업 전용
야간비행(혼디 대표)과 이경선(디퍼살롱 리더)이 함께 진행하는 협업 채널이야. 현재 협업 범위: - 혼디 트리비움 집필 공동 작업 - 강의 기획 및 디퍼살롱과의 연계 프로그램 - 기획 제안서 검토 및 보완 - 디퍼 프레임워크를 혼디 프로젝트에 적용하는 방법 이 대화에서: - 두 사람이 각자 가져온 내용을 통합하는 방향으로 도와줘 - 이경선의 트리비움 전문성과 야간비행의 제주 현장 경험이 시너지가 나도록 - 결과물은 언제나 혼디의 언어와 디퍼의 방법론이 함께 살아있어야 해
Trivium × Hondi

트리비움을 혼디에
어떻게 적용하는가

디퍼에서 배우는 트리비움(문법→논리→수사)이 혼디 콘텐츠 작업의 방법론이 됩니다. 매달 하나씩 레이어가 쌓입니다.

5월 — 문법 (Grammar): 혼디의 원천 언어 발굴

디퍼 일간 질문을 탐구하면서 동시에 혼디의 핵심 언어를 쌓습니다.

5월 이중 작업: 매일 질문으로 "나"를 해체하는 동시에 — "이 발견이 혼디에서는 어떻게 말해질까?"를 HONDI/01-원천텍스트/에 따로 기록합니다. 두 노트가 서로를 풍요롭게 합니다.
5월 혼디 탐구 프롬프트 — "혼디 프로젝트" 에서
오늘 디퍼 탐구에서 나온 발견이야: [Daily Note의 "Claude와 탐구한 것" 섹션 붙여넣기] 이것을 혼디 프로젝트의 언어로 번역한다면? "제주에서 함께 산다"는 문맥에서 이 발견이 어떤 의미를 갖는지 탐구해줘. 혼디의 원천 텍스트 파일에 추가할 만한 문장이 있다면 제안해줘.

6월 — 논리 (Logic): 혼디 구조 설계

5월에 쌓인 원천 텍스트로 집필 목차, 강의 구조, 기획 제안서 프레임을 만듭니다.

  • 집필: HONDI/02-집필/ — 원천 텍스트에서 책 목차 초안 도출. 트리비움 3부 구조 활용 가능
  • 강의: HONDI/03-강의/ — 혼디가 가르칠 수 있는 것의 커리큘럼 설계
  • 기획 제안: HONDI/04-기획제안/ — 원천 텍스트에서 나온 나다운 언어로 제안서 작성
6월 구조화 프롬프트 — "혼디 프로젝트"에서
지금까지 5월에 쌓인 혼디 원천 텍스트야: [HONDI/01-원천텍스트/혼디-핵심언어.md 내용 붙여넣기] 이것을 토대로: 1. 책 목차의 가능성 — 트리비움 3부 구조로 잡는다면? 2. 강의 커리큘럼 — 누구에게, 무엇을, 어떤 순서로? 3. 기획 제안서 한 장 — 혼디의 언어로 간결하게 세 가지 방향 모두 초안을 제안해줘.

7월 — 수사 (Rhetoric): 혼디의 목소리로 세상에

기획 제안서를 보내고, 강의 첫 회를 열고, 집필 챕터를 공개합니다.

  • 기획 제안서 → 실제 발신
  • 강의 커리큘럼 → 첫 강의 일정 확정
  • 집필 챕터 1개 → 블로그·뉴스레터·SNS 중 하나에 퍼블리시
